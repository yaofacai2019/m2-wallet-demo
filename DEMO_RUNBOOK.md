# M2 Wallet 内部 Demo 演示手册

更新日期：2026-07-23

## 启动

在项目目录运行：

```bash
python3 -m backend.server
```

浏览器打开 `http://127.0.0.1:8787`。

本 Demo 只绑定本机地址，使用模拟签名和模拟广播，不会发送真实数字资产。

## 演示账号

浏览器沙箱可选择下列角色，并输入任意非空沙箱密码。若运行 Python 后端，
请先通过 `.env.example` 所列环境变量配置各角色密码。

| 角色 | 账号 | 密码 | 主要用途 |
| --- | --- | --- | --- |
| 管理员 | `admin` | 环境变量配置 | 查看全部功能、审批、风控和审计 |
| 运营 | `operator` | 环境变量配置 | 创建支付订单和提现申请 |
| 财务 | `finance` | 环境变量配置 | 审批或拒绝提现 |
| 只读 | `viewer` | 环境变量配置 | 只查看数据，不能创建或审批 |

## 路径一：客户使用稳定币支付

1. 使用运营账号登录。
2. 打开“支付引擎”，进入支付项目。
3. 点击“创建订单”，选择 `USDT · TRON` 或 `USDC · Polygon`。
4. 打开新订单详情，点击“预览客户收银台”或“打开独立收银台”。
5. 独立支付链接格式为 `/pay/{M2订单ID}`，客户无需登录后台即可查看应付金额、网络和唯一支付地址。
6. 点击“模拟客户已完成付款”。
7. 独立收银台会自动轮询订单状态；到账后显示“支付完成”。返回订单列表也会看到状态和交易哈希。
8. 打开“结算记录”，查看平台手续费和净结算金额；USDT 与 USDC 分开结算。

### 异常支付验收

在支付订单详情中可直接选择 `Simulate underpayment`、`Simulate full payment`、`Simulate overpayment` 或 `Mark expired`。列表和客户收银台会同步显示 Underpaid、Paid、Overpaid、Expired，并保留应付金额与实付金额。

创建订单时可填写客户平台的 Merchant order ID、Customer ID、Return URL 和 Metadata。完整对接字段及签名回调格式见 `INTEGRATION_CONTRACT.md`。

## 路径二：支付地址归集

1. 使用管理员或运营账号登录。
2. 打开 `Risk Control → Automation`。
3. 在 `Assets awaiting sweep` 中查看达到阈值的 USDT 或 USDC 子地址余额。
4. 点击 `Sweep now`，系统创建归集任务、生成模拟交易哈希并写入复式账本。
5. 在 `Sweep history` 中打开任务详情，并在 `Transactions` 中查看对应的内部划转。
6. 归集完成后，钱包资产页的 `Sweepable` 金额归零。

## 路径三：客户提 U，财务确认后自动打款

1. 使用运营或管理员账号登录。
2. 打开“钱包”，点击“发送”。
3. 选择 USDT、TRON，填写接收地址和金额。
4. 点击“提交财务审批”，申请会进入“协作 → 待审批”。
5. 切换财务账号，打开待审批详情。
6. 点击“审批通过”。
7. 系统自动完成模拟签名、模拟广播、交易确认和交易哈希回写。
8. 在“协作 → 已审批”、“WaaS 项目 → 钱包提币”和“交易记录”查看同一业务单。

## 路径四：权限和风控

1. 使用只读账号尝试创建订单，创建入口不会出现。
2. 使用运营账号尝试审批提现，后端会拒绝。
3. 使用管理员账号打开“风控 → 策略”，可暂停全部出款或修改单笔限额。
4. 使用管理员或财务账号打开“管理 → 审计日志”，查看登录、创建、审批和策略修改记录。

## 路径五：WaaS 项目设置

1. 使用管理员账号打开 `WaaS → Project settings`。
2. 在 Developer 中查看 API Key、支持网络和集成说明。
3. 在 IP Allowlist 中配置允许访问 API 的来源 IP 或 CIDR。
4. 在 Callback settings 中配置客户平台回调 URL、最低回调金额及提现外部验证开关。
5. 关闭项目后，新建支付和提现请求会被后端阻止；重新启用后恢复。

## 路径六：商户回调沙箱

1. 使用管理员账号打开 `WaaS Projects → Settings → Callback settings`。
2. 点击 `Use demo receiver`，把项目回调地址指向本机商户接收器。
3. 创建并完成一笔新支付订单，或批准一笔新提现。
4. 打开 `Callback history`，点击 `Deliver pending callbacks`。
5. 页面显示已验签回执数量，回调状态更新为 `Delivered`。
6. 沙箱使用与真实商户相同的时间戳和 HMAC-SHA256 验签；HTTP 仅允许本机回环地址。

## 路径七：提现外部验证

1. 在 `WaaS Projects → Settings → Callback settings` 开启 `External withdrawal validation`。
2. 创建一笔提现申请并由财务账号打开审批详情。
3. 点击批准后，M2 Wallet 先向客户平台发送签名校验请求。
4. 客户平台批准时，系统继续签名、广播和确认；审批时间线显示 External validation。
5. 客户平台拒绝时，系统不签名、不广播，申请保留在待审批并记录拒绝原因。
6. 本机 Demo 默认使用商户沙箱校验器；正式环境通过 `M2_WITHDRAWAL_VERIFICATION_URL` 配置 HTTPS 端点。

## 路径八：白名单和每日限额

1. 使用管理员账号打开 `Risk Control → Policies`。
2. `Per-transaction limit` 显示并控制单笔出款上限。
3. `Recipient allowlist` 可启用或关闭；启用后，未知接收地址无法创建提现。
4. `Daily cumulative payouts` 展示当天已发送金额与日限额。
5. 当本笔金额加当天累计金额超过限额时，财务审批无法进入签名阶段。
6. 修改策略会写入审计日志；默认日限额可通过 `M2_DAILY_WITHDRAWAL_LIMIT` 配置。

## 路径九：一键验收

1. 使用管理员或财务账号打开 `Management → Demo Acceptance`。
2. 页面实时读取数据库和账本，检查八个交付领域。
3. 检查范围包括稳定币收付、异常订单、归集、财务出款、商户验证、签名回调、风控和对账。
4. 点击 `Refresh evidence` 可重新计算，不依赖缓存或人工填写。
5. `8 of 8 checks passed` 且对账为 Balanced 时，表示当前内部 Demo 数据满足演示验收条件。

## 注意

- 页面里的收款地址、签名、广播和交易哈希均为 Demo 数据。
- 不要在项目、聊天或浏览器中填写真实助记词、私钥或签名密钥。
- 接入测试链或主网前，必须改用独立签名服务，并重新验收限额、白名单、余额和回调策略。
