# M2 Wallet 稳定币收付 Demo

## 当前可交互原型

在项目目录执行 `python3 -m backend.server`，然后访问
`http://127.0.0.1:8787`。Demo 使用本机 SQLite 数据库，不连接真实钱包，
也不会广播真实资产交易。

原型已包含：

- 创建 USDT / USDC 收款订单
- 为订单生成唯一收款地址
- 模拟链上到账和状态更新
- 按订单费率计算平台佣金
- 提 U 审批队列
- 财务确认后模拟锁定、签名、广播和交易哈希回写
- 批量处理小额提现
- 钱包余额与出款策略展示
- 客户平台收款和提现 API 示例
- SQLite 持久化 Demo 操作状态、复式账本、审计日志和风控设置
- 管理员、运营、财务和只读四类角色权限
- USDT 与 USDC 分币种佣金、净结算和结算详情

## 两周实现范围

### 第 1–3 天：业务与数据

- 收款订单、提现单、钱包和账本数据表
- 幂等的创建收款 / 提现 API
- 商户 API Key 和回调签名
- 内部管理员登录与权限

### 第 4–6 天：真实收款

- TRON USDT 地址分配
- TRON 区块扫描和 TRC20 Transfer 识别
- 确认数、重复事件和金额精度处理
- 到账记账、佣金记账和支付回调
- Polygon 测试网 USDT / USDC 适配

### 第 7–9 天：真实提现

- 提现地址校验
- 余额锁定和双花防护
- 财务审批和审计日志
- 独立签名服务接口
- 交易构建、广播、确认和回调
- TRON 能量 / TRX 余额预警

### 第 10–12 天：账本与异常

- 简化复式账本
- 订单、链上交易和钱包余额对账
- 回调重试
- 交易失败、超时和人工复核
- 总暂停开关和限额

### 第 13–14 天：联调与验收

- 对接客户平台测试环境
- 跑通收款、佣金、提现审批和出款全链路
- 重复请求、重复回调和余额不足测试
- 部署内部 Demo 环境

## 两周 Demo 不包含

- 自研 MPC
- 冷钱包和灾难恢复体系
- 主网大额资金
- 自动退款、兑换和法币结算
- Bitcoin、Solana 等其他账户模型
- 完整商业计费和 SLA 体系
